# RMG FLUX.2 Experiments

This folder contains the FLUX.2 version of the follow-the-mean idea from the paper: Reference-Mean Guidance (RMG) applied at inference time to a frozen pretrained text-to-image flow model.

The smaller `moons/` and `mnist/` experiments make the mechanism visible in controlled settings. This folder asks the same question in a real image model:

Can a pretrained generator be guided toward a concept, style, object, or pose just by giving it examples, without fine-tuning, auxiliary reward models, classifier gradients, or test-time search?

The answer explored here is RMG. Build a small reference set, encode it into the model's latent space, and during sampling add a velocity correction toward the empirical reference endpoint mean. The text prompt still provides the base generation request. The reference set changes the endpoint mean, and changing the endpoint mean changes the flow.

In the paper's shorthand: guide with examples, not rewards.

## Core Idea

RMG starts from a reference set:

```text
R = {reference image 1, reference image 2, ..., reference image M}
```

The references can come from prompts generated by the same FLUX.2 model, or from real images on disk. Each reference image is encoded into VAE latents. During generation, the sampler has a current latent state `x_t`. RMG compares that state to the reference latents and computes an empirical reference endpoint mean:

```text
mu_ref(x_t, t) = empirical reference endpoint mean
```

The guidance velocity points from the current state toward that mean:

```text
v_guided(x_t, t) = (mu_ref(x_t, t) - x_t) / (1 - t)
```

The callback adds this correction to the model's normal denoising trajectory:

```text
latents <- latents + beta_t * v_guided * dt
```

The scalar `beta_t` is the guidance schedule. In these experiments it can be constant, bell-shaped, or quadratic-decay. This is the main control knob: larger values make the reference endpoint mean pull harder; the start/end fractions decide which part of the denoising trajectory receives guidance.

No FLUX.2 weights are updated. The method is training-free and uses one sampling trajectory per image.

## Experiment Story

The configs in `experiments/` are meant to be read as a progression from single examples to controlled sweeps.

### 1. Prompt Plus Reference Set

Single-case configs test the basic behavior: keep the prompt broad, then use the reference set to inject the missing attribute.

Examples:

```text
pink_elephant.yaml          prompt: an elephant in a jungle        reference: a pink elephant
blue_elephant.yaml          prompt: an elephant in a jungle        reference: a blue elephant
cat_van_gogh.yaml           prompt: a cat                          reference: a van gogh painting
house_forest_sketch.yaml    prompt: a house in a forest            reference: a pencil sketch
animal_savanna_giraffe.yaml prompt: an animal in a savanna         reference: a giraffe
```

Each run saves a baseline image, an RMG-guided image, the nearest reference neighbor, a comparison strip, callback statistics, and JSON metadata.

What to look for:

- The baseline shows what the prompt alone produces.
- The RMG-guided image shows how the reference set changes the output.
- The nearest reference neighbor is a sanity check: guidance should use the reference set without simply copying one image.

### 2. Image Reference Sets

Some configs use real image folders instead of prompt-generated references.

Examples:

```text
hand_horns.yaml        reference images: references/horns
gymnast_ring_leap.yaml reference images: references/ring-leap
```

These runs test a harder kind of control. The reference set may encode pose or structure that is awkward to specify with text alone and difficult to score with a simple scalar reward. The method still uses the same machinery: resize/pad images, encode them to latents, compute the reference endpoint mean during sampling.

### 3. Prompt-Reference Interaction

`ablation_prompt_reference_pink_elephant.yaml` crosses two prompts with two reference sets:

```text
neutral prompt      x neutral reference
neutral prompt      x attribute reference
attribute prompt    x neutral reference
attribute prompt    x attribute reference
```

This experiment asks whether the prompt and reference set act as separable controls. The prompt says what the image should be about; the reference set biases the visual direction.

### 4. Reference Composition

`ablation_reference_composition.yaml` changes the composition of the reference set.

For example, in the savanna-object case, the prompt stays:

```text
an animal in a savanna
```

The reference set gradually changes from zebra-heavy to giraffe-heavy. The generated outputs should track that composition. This is the high-dimensional version of the composition story from `moons/`: if the reference endpoint mean changes, the flow changes.

### 5. Guidance Schedule And Strength

`ablation_betas.yaml` sweeps:

```text
constant
bell
quadratic-decay
```

and a range of guidance strengths. This experiment asks when RMG should intervene. Too little guidance may not move the sample; too much can overpower the base model or create artifacts. The schedule controls where the reference set has influence along the trajectory.

### 6. Reference Size And Sampling Steps

`ablation_dataset_size.yaml` runs `ablate_reference_size.py`, varying the subset size `M`. It evaluates CLIP target similarity, optional VLM success, and LPIPS diversity.

`ablation_nfe.yaml` runs `ablate_sampling_steps.py`, varying the number of inference steps and guidance strengths for the ring-leap image-reference case.

These are the reliability checks:

- How many references are enough?
- Does the method still work with fewer sampling steps?
- When does guidance start improving the target concept versus adding artifacts?


`references/` is intentionally ignored by git except for `.gitkeep`. The remote repository keeps the folder empty; generated caches and local image reference sets live there on the machine that runs the experiments.

## Run

From the repository root:

```bash
cd rgm
./run.sh experiments/pink_elephant.yaml
```

Use a dry run to validate the config and print the resolved command:

```bash
./run.sh experiments/pink_elephant.yaml --dry-run
```

Direct Python usage is also config-driven:

```bash
python src/training_free_control.py --config experiments/pink_elephant.yaml
python src/ablate_reference_size.py --config experiments/ablation_dataset_size.yaml
python src/ablate_sampling_steps.py --config experiments/ablation_nfe.yaml
```

The launcher resolves paths relative to `rgm/`, validates the YAML, stages any listed references into the run directory, and executes the selected script.

## Config Schema

A single prompt-reference case looks like this:

```yaml
name: pink_elephant
script: training_free_control.py
sections:
- case
common:
  out_dir: runs/pink_elephant
retrieval:
  reuse_reference: true
  reference_size: 20
  guidance_strength: 0.2
  beta_schedule: bell
  guidance_start_frac: 0.15
  guidance_end_frac: 0.95
  topk: 4
  callback_verbose: false
case:
  slug: pink_elephant
  prompt: an elephant in a jungle
  reference_set:
    type: prompts
    label: pink elephant reference
    prompt: a pink elephant
stage_references:
- source: a-pink-elephantx20-da106fd69346
```

For image references, switch the reference set type:

```yaml
case:
  slug: hand_horns
  prompt: a hand doing the sign of the horns
  reference_set:
    type: images
    label: horns image reference
    image_dir: references/horns
```

Important retrieval fields:

- `reference_size`: number of references to generate or use.
- `reuse_reference`: reuse an existing reference cache if present.
- `guidance_strength`: base value for the guidance schedule.
- `beta_schedule`: one of `constant`, `bell`, or `quadratic-decay`.
- `guidance_start_frac` / `guidance_end_frac`: denoising window where RMG is active.
- `topk`: restrict the empirical reference mean to the top-k closest references.

## Outputs

Single-case runs write outputs under the configured `common.out_dir`, for example:

```text
runs/pink_elephant/
```

Ablation runs write a generation manifest before evaluation starts. This matters because VLM/CLIP evaluation can be expensive and fragile; generation outputs are checkpointed to disk first.

## Evaluation Helpers

The RMG generation scripts save metadata and image paths. The evaluation helpers reuse those outputs.

```bash
python src/evaluate_prompt_reference_clip.py \
  --input-dir runs/ablation_prompt_reference_pink_elephant/prompt_reference_interaction

python src/evaluate_savanna_clip_sweep.py \
  --input-dir runs/ablation_reference_composition/controllability/savanna_object \
  --out-dir runs/savanna_object_clip_sweep

python src/evaluate_savanna_object_vlm.py \
  --input-dir runs/savanna_object_clip_sweep
```

CLIP scores are used for continuous similarity checks. VLM prompts are used for simple yes/no or zebra/giraffe classification when the experiment needs a semantic success rate.

## Environment Notes

These scripts load FLUX.2 through `diffusers` and evaluation models through `transformers`. The main generation path expects a GPU for practical runtimes.

The default model id is:

```text
black-forest-labs/FLUX.2-klein-4B
```

If a run fails at model loading, check:

- access to the model on Hugging Face,
- a recent enough `diffusers` install for `Flux2KleinPipeline`,
- available GPU memory,
- `HF_HOME` / Hugging Face cache settings on the machine.

## Why This Experiment Matters

`moons/` shows that shifting a weighted endpoint mean can steer a flow in two dimensions. `mnist/` shows the same idea on simple images. `rgm/` asks whether the mechanism survives contact with a large pretrained image generator.

That makes this folder the training-free, pretrained-model endpoint of the repository:

```text
two moons -> MNIST -> RMG with FLUX.2
```

The important constraint is that RMG does not train a new model. It treats the pretrained generator as a base flow and injects reference-set information only during sampling. This makes the experiment useful for studying a clean question: how much control is already available if we can choose the references and follow their endpoint mean?
